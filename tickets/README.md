# Tickets

In most software teams, work is broken down into **tickets** — small, self-contained tasks that describe something to build, fix, or improve. You might hear them called issues, stories, or cards depending on the team.

Each ticket in this folder is one task for you to complete on the Eurostar Timetable project. They are ordered roughly from easiest to hardest.

---

## How to work a ticket

1. Read the ticket fully before touching any code.
2. Find the file(s) mentioned in **Files to edit**.
3. Look for the `// TODO: EURO-XX` comment that marks the right spot.
4. Make the change, save the file, and refresh the browser to see the result.
5. Check the **Acceptance criteria** — does everything listed work correctly?
6. Ask a developer if you are stuck.
